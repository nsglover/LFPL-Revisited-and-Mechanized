\section{Polynomial-Time Soundness}
\label{sec:soundness}

This section shows that \lfpl programs can be evaluated in polynomial
time.
%
To strengthen the soundness result and illustrate how to add features to \lfpl while maintaining polynomial-time soundness, we extend the language and refer to the result as \lfplplus. The features are the standard product type, which manifests as the lazy product in an affine setting, a type of unbounded stacks which are similar to lists but do not require diamonds to construct and do not support recursion, and the inductive type of binary trees. The syntax and typing rules of these language constructs are given in \pref{fig:ext-syntax-tp-tm} and \pref{fig:ext-jmt-ty} respectively.

\input{figures/ext-syntax-tp-tm}
\input{figures/ext-jmt-ty}

The typing rules for stacks of type $\tpstack{A}$ correspond to those for lists in a standard non-affine functional programming language, where the elimination from is case analysis rather than structural recursor and construction does not require diamonds.
%
These stacks do not violate \lfpl's core principle of controlling recursion by diamonds because they do not support recursion.

The typing rules for the tree type $\tptree{A}$, an inductive type which does support recursion, are similar to those of lists $\tplist{A}$. The primary difference is that, in \pref{rule:ty-TreeE}, the base case $N_1$ is typed under an empty context. Unlike lists, where there is only ever one terminal $\tmnil$, $N_1$ needs to be run for each leaf and allowing access to a context potentially containing diamonds would allow for the duplication of diamonds.

The denotational semantics for these new constructs are relatively straightforward. We set $\tpsem{\tpprod{A}{B}} = \tpsem{A} \times \tpsem{B}$ and $\tpsem{\tpstack{A}} = \tpsem{A}^*$. Giving a semantics for $\tptree{A}$ and its related terms boils down to giving a mathematical account of binary trees (in the same manner we have used the Kleene star to give a simple mathematical account of lists).
%
The semantics for terms involving these types can be found in the mechanization.

The stack type $\tpstack{A}$ is a particularly powerful addition to \lfpl. The vast majority of the difficulty in proving polynomial-time completeness of \lfpl, and by far the most complex aspect of our entire mechanization, was the definition and implementation of the bounded stack data structure discussed in \pref{sec:completeness-bounded-stack-definition} and \pref{sec:completeness-bounded-stack-implementation}. The addition of unbounded, $\tpdiam$-free stacks to \lfpl makes all of this unnecessary; \lfplplus admits a much simpler completeness proof.



\subsection{Big-Step Operational Cost Semantics}
\label{sec:operational-semantics}

To reason about the cost of evaluating \lfplplus programs, we need reason about evaluating functions which may take lists as input. For example, we have seen a closed \lfpl term $\code{reverse} : \tparr{\tplist{A}}{\tplist{A}}$, but we can never actually ``run'' $\code{reverse}$ on a nonempty input list, since that would require a closed term of type $\tpdiam$ and the whole point of \lfpl's type system is to make that impossible.

To solve this problem, we introduce a separate language for \lfplplus values (see \pref{fig:syntax-val}). Note the definitions of value and environment are mutually recursive. This implies that we usually have to define functions on values via mutual recursion with a corresponding function on environments. Also, note that there is a value $\vdiam$ of type $\tpdiam$, and there are also nonempty list and tree values which do not require diamonds to construct. \comment{Nathan}{I don't understand the previous remark about not requiring diamonds.}
%
By keeping this value language separate from the term language of \lfplplus, we ensure \lfplplus terms are still polynomial-time sound while still obtaining a means of giving \lfplplus programs concrete inputs to help us observe their evaluation. We also define a typing judgement $v : A$ for values, so that $\vdiam : \tpdiam$, $\vcons{v_h}{v_t} : \tplist{A}$ whenever $v_h : A$ and $v_t : \tplist{A}$, and so on.

\input{figures/syntax-val}

To account for the cost of evaluation, we give a cost-annoted big-step operational semantics, represented by the judgement $\evals{\env}{M}{c}{v}$, read ``term $M$ evaluates under the environment $\env$ to a value $v$ with cost $c$''. A representative set of rules for this judgement is given in \pref{fig:jmt-eval}, and a complete set of rules can be found in \pref{app:evaluation}. Note the presence a constants $\Cvar, \Cnull, \ldots, \Ctrec \in \mathbb N$ (one for each syntactic construct) in the conclusion of each rule. These are left arbitrary so that our cost model is generic with respect to the particular costs of each primitve operation of the language.

\input{figures/jmt-eval}

This semantics enjoys standard properties such as determinism (there is at most one value and cost that a given program may evaluate to), preservation of typing (a well-typed expression evaluates to a value with the same type, assuming all values in the environment have the correct type), and equivalence with the denotational semantics, assuming a sensible denotation of values. We have mechanized both of these results, but neglect stating or proving them here because they are straightforward and not relevant to the discussion of polynomial-time soundness.



\subsection{The Non-Size-Increasing Property}

We now finally make precise the non-size-increasing property referred to in previous sections. We will formulate size in terms of the operational semantics, whereas the original formulation was in terms of a set-theoretic denotational semantics \cite{hofmannLinearTypesNonsizeincreasing1999}. The operational semantics simplify the formulation while still supporting a reasonable soundness theorem.

Our goal is to assign to each value $v$ a number $\valsize{v} \in \mathbb N$ that, roughly speaking, counts the number of diamond values $\vdiam$ contained within $v$. We then want to show that feeding values to \lfplplus terms does not increase their size (i.e. the size of the output value is bounded by the size of the inputs). With goal in mind, we define a mutually inductive size function on values and environments in \pref{fig:size-val-env}.

\input{figures/size-val-env}

Our intuitive notion of the non-size-increasing property is that a function's output should have size no greater than its input. We are finally in a position to make this rigorous:
\begin{theorem}[The Non-Size-Increasing Property]
\label{thm:non-size-increasing}
Suppose that $\types{\ctx}{M}{A}$. Then, for all environments $\env$, values $v$, and costs $c$ such that $\evals{\env}{M}{c}{v}$, we have $\valsize{v} \le \envsize{\env}$.
\end{theorem}
\begin{proof}
By rule induction on $\evals{\env}{M}{c}{v}$.
\end{proof}

The relevance of this result to soundness is already clear; an immediate corollary is that there are no closed terms of type $\tpdiam$, one of the main properties \lfpl's type system was designed to ensure. Indeed, if there were such a term that evaluates (we shall see in the next section that all terms evaluate), it would have to evaluate to $\vdiam$ which has size $1$, yet the empty environment has size $0$.



\subsection{Polynomial Cost Bound}

Our goal in this subsection is to prove the following:

\begin{theorem}[Concrete Polynomial-Time Soundness]
\label{thm:soundness-concrete}
Suppose $\types{\ctxcons{\ctxnil}{x}{\tplist{A}}}{M}{B}$, where $\dfree{A}$. Then, there exists a polynomial $P_M : \mathbb N \to \mathbb N$ such that, for any value $v_x : \tplist{A}$ made up of $n$ nodes (i.e. the list represented by $v$ has length $n$), there exists a value $v : B$ and a cost $c \in \mathbb N$ such that $\evals{\envcons{\envnil}{x}{v_x}}{M}{c}{v}$ and $c \le P_M(n)$.
\end{theorem}

It is important that the polynomial only depends on $M$, i.e. it is quantified before $\env$. Otherwise, we could just do cost-agnostic termination proof via standard methods to obtain $v$ and $c$, and then take $P_M(n) = c$, trivializing the proof and not saying anything useful about \lfplplus.

Taking $B = \tplist{\tpunit}$ and $A = \tpunit$, we recover the basic notion of soundness for \lfplplus functions on natural numbers. We present this more general form to motivate our transition to the even more general form we need to successfully prove it. As usual, we must reason about arbitrary terms under arbitrary contexts. It is tempting to say the following:

\begin{antitheorem}[General Soundness - First Attempt]
Suppose $\Gamma \vdash M : A$. Then, there exists a polynomial $P_M : \mathbb N \to \mathbb N$ such that, for any $\env : \Gamma$, there exists a value $v : A$ and a cost $c \in \mathbb N$ such that $\evals{\env}{M}{c}{v}$ and $c \le P_M(\envsize{\env})$.
\end{antitheorem}

This is a step in the right direction but is blatantly false; the problem is that the environment may contain values which incur cost when used, which is dynamic information and $P_M$ only has access to static information. Concretely, consider $\types{f : \tparr{\tpunit}{\tpunit}}{\tmapp{f}{\tmnull}}{\tpunit}$. If for simplicity $\Capp = \Cvar = \Cnull = 0$, we cannot statically observe how this program could incur any cost. After all, the only statically observable syntactic constructs are application, variable, and null. On the other hand, dynamically, $\env$ could map $f$ to a function which incurs nonzero cost.

To resolve this issue, we must consider the potential cost of running functions (or lazy pairs) stored in the environment $\env$ and result value $v$. Suppose for a moment have defined three families of polynomials: $P_M$ on all terms $M$, $P_v$ on all values $v$, and $P_\env$ on all environments $\env$. Our new goal is:

\begin{theorem}[General Soundness - Second Attempt]
\label{thm:soundness-general-weak}
Suppose $\Gamma \vdash M : A$. Then, for any $\env : \Gamma$, there exists a value $v : A$ and a cost $c \in \mathbb N$ such that $\evals{\env}{M}{c}{v}$ and:
$$c + P_v(\envsize{\env}) \le P_M(\envsize{\env}) + P_{\env}(\envsize{\env})$$
\end{theorem}

This theorem is true, though as stated it is not strong enough to be proven by rule induction on $\types{\ctx}{M}{A}$. Before getting into that, let's define the polynomials used in this theorem.

\begin{definition}[Polynomial Algebra]
We define $\max(P,Q) : \mathbb N \to \mathbb N$ as the polynomial with coefficients equal to the pairwise maximums of the coefficients of polynomials $P,Q : \mathbb N \to \mathbb N$. This way, $\max(P(n), Q(n)) \le \max(P,Q)(n)$. Additionally, we define $X$ to be the identity polynomial, i.e. $X(n) = n$.
\end{definition}

\begin{definition}[Term Polynomial]
The aim of the polynomial $P_M : \mathbb N \to \mathbb N$ is to bound the cost of evaluating $M$, and the future cost of using any functions (or lazy pairs) it might evaluate to, by $P_M(n)$ in an environment containing $n$ diamonds.
\begin{align*}
P_{x} &= \Cvar \\
P_{\tmrecord{M_1}{M_2}} &= \Crecord + \max(P_{M_1}, P_{M_2}) \\
P_{\tmproj{i}{M}} &= \max(1, \Cproj{i}) + P_M \\
P_{\tmlam{x}{M}} &= \Clam + P_M \\
P_{\tmapp{M}{N}} &= \max(1, \Capp) + P_M + P_N \\
P_{\tmcase{M}{x_1}{N_1}{x_2}{N_2}} &= \Ccase + P_M + \max(P_{N_1}, P_{N_2}) \\
P_{\tmrec{M}{N_1}{x_d}{x_h}{x_t}{N_2}}(n) &= P_M + (\Crec + P_{N_1}) \;  + \\
&\quad\quad X * (\Cvar + \Crec + P_{N_2}) \\
P_{\tmtrec{M}{N_1}{x}{y}{z}{w}{N_2}} &= P_M + (X + 1) * (\Ctrec + P_{N_1}) \; + \\
&\quad\quad X * (2\Cvar + \Ctrec + P_{N_2})
\end{align*}
We leave $P_M$ for other terms $M$ to the reader.
\end{definition}

The intuition behind $P_{\tmrec{M}{N_1}{x_d}{x_h}{x_t}{N_2}}(n)$ is that $M$ and $N_1$ are evaluated once, and $N_2$ could be evaluated up to $n$ times, where $n = \envsize{\env}$ is the number of diamonds available the environment. Similarly, for $P_{\tmtrec{M}{N_1}{x}{y}{z}{w}{N_2}}$, the base case $N_1$ could be evaluated $n + 1$ times (a binary tree with $n$ internal nodes has $n + 1$ leaf nodes) and $N_2$ could be evaluated $n$ times.

The weakening of the bound to $\max(1, \Capp)$ in $P_{\tmlam{x}{M}}$ instead of just $\Capp$ (and similarly in $P_{\tmproj{i}{M}}$) is necessary for how we approach the soundness proof; it's purpose will become evident in \pref{sec:proving-general-soundness}

\begin{definition}[Value and Environment Polynomials]
The idea of $P_v$ and $P_\env$ is to bound the cost of evaluating all of the function bodies and lazy pairs stored inside $v$ and $\env$ respectively. We define the two at once via mutual induction.
\begin{align*}
P_{\vdiam} &= P_{\vnull} = P_{\vempty} = P_{\vnil} = P_{\vleaf} = 0 \\
P_{\vrecord{\env}{M_1}{M_2}} &= P_{\env} + \max(P_{M_1}, P_{M_2}) \\
P_{\vlam{\env}{x}{M}} &= P_{\env} + P_M \\
P_{\vpair{v_1}{v_2}} &= P_{v_1} + P_{v_2} \\
P_{\envnil} &= 0 \quad\text{and}\quad P_{\envcons{\env}{x}{v}} = P_\env + P_v
\end{align*}
We leave $P_v$ for other values $v$ to the reader.
\end{definition}

Notice with these choices of polynomials that \pref{thm:soundness-general-weak} implies \pref{thm:soundness-concrete}, because $P_v = 0$ and $\valsize{v} = n$, where $n$ is the length of the list represented by $v$, when $v : \tplist{A}$ and $\dfree{A}$.



\section{Proving the Theorem}
\label{sec:proving-general-soundness}

\pref{thm:soundness-general-weak} contains a statement that every term $\types{\ctx}{M}{A}$ terminates under an environment $\env : \ctx$. We prove this in tandem with the polynomial cost bound to make use of the fact that the value of the term polynomial $P_M$ is strictly increasing at the most difficult points of a standard termination proof -- the elimination forms of negative types. Specifically, we defined $P_{\tmapp{M}{N}} = \max(1, \Capp) + P_M + P_N$ so that $P_{\tmapp{M}{N}}(n) > P_M(n) + P_N(n)$ for all $n$, and similarly for the lazy product type. We can take advantage of this strict difference via strong induction on our cost bound.

Another necessary strengthening of \pref{thm:soundness-general-weak} is break the dependency of the polynomials' input on the environment $\env$. Instead, we can just prove the theorem holds for all inputs $n$ which are big enough to account for all diamonds, i.e. $n \ge \envsize{\env}$. The final form of our soundness theorem is as follows:

\begin{theorem}[General Polynomial-Time Soundness]
For all $k \in \mathbb N$, suppose that $\types{\ctx}{M}{A}$ and $\env : \ctx$. If $n \ge \envsize{\env}$ and $k \ge P_M(n) + P_{\env}(n)$, then there is exists a value $v : A$ and a cost $c \in \mathbb N$ such that $\evals{\env}{M}{c}{v}$ and:
$$c + P_v(n) \le P_M(n) + P_{\env}(n)$$
\end{theorem}
\begin{proof}
While $k$ may seem to have no real involvement in the theorem statement, it is the crucial means by which we reason about the soundness of terms that arise dynamically from function and lazy pair closures in the environment. We begin with a strong induction on $k$, so we know the theorem holds for all $k' < k$.

Next, we go by rule induction on $\types{\ctx}{M}{A}$. At the cases for $\tmapp{M}{N}$ and $\tmproj{i}{M}$, where we have designed the polynomial for these terms to be strictly larger than the polynomials for its subterms, we will be able invoke our outer induction hypothesis about $k$ to reason about, say, the term $M'$ where $\evals{\env}{M}{c}{\vlam{\env'}{x}{M'}}$.

\pref{thm:non-size-increasing} comes up in nearly every case, since we need to satisfy the premise $n \ge \envsize{\env}$ of any inductive hypotheses. Importantly, we actually make use of this premise in the list and tree recursor cases, where we can say for example that the length of any list value $v$ is at most $\envsize{\env}$ and therefore at most $n$.

Most cases are verbose but straightforward, and a couple of the less straightforward cases can be found in \pref{app:soundness}.
\end{proof}
